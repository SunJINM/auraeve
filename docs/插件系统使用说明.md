# AuraEve 插件系统使用说明

本文仅覆盖插件（Plugin）能力。`clawhub` 发布的是 skill，不属于插件安装来源。

## 1. 插件定义

插件是可加载的 Python 扩展，最小结构如下：

```text
my-plugin/
  auraeve.plugin.json
  plugin.py
  skills/
    SKILL.md
```

最小 `auraeve.plugin.json` 示例：

```json
{
  "id": "demo.local",
  "entry": "plugin.py",
  "name": "Demo Local",
  "version": "0.1.0",
  "skills": ["skills"]
}
```

## 2. 安装位置与状态

- 插件安装目录：`~/.auraeve/extensions/<plugin_id>`
- 插件状态文件：`~/.auraeve/plugins-state.json`
- 可通过 `AURAEVE_STATE_DIR` 改写状态根目录

## 3. CLI 命令

```bash
python main.py plugins list --json
python main.py plugins info <id> --json
python main.py plugins install <path> [--link]
python main.py plugins enable <id>
python main.py plugins disable <id>
python main.py plugins uninstall <id> [--keep-files]
python main.py plugins doctor --json
```

说明：
- `plugins install` 只支持本地目录或压缩包路径。
- 不支持 `clawhub:*` 或 `https://clawhub.ai/...`。
- 安装/启停/卸载后建议重启 AuraEve 进程。

## 4. WebUI 使用

插件页支持：
- 列表与详情
- 安装（本地路径）
- 启用/禁用
- 卸载
- 诊断

安装输入框只接受本地插件路径（目录或压缩包）。

## 5. 插件与 skill 的关系

- 插件可在 manifest 的 `skills` 字段声明技能目录。
- 只有启用状态的插件，其声明的 skills 才会参与技能加载。
- `clawhub` 上的是 skill 分发，不是插件分发。

## 6. 常见问题

1. `auraeve.plugin.json not found`
- 目录或压缩包内缺少 manifest，或路径错误。

2. `path not found: ...`
- 输入路径不存在，确认绝对路径和访问权限。

3. `plugins install` 提示不支持 clawhub
- 这是预期行为。请改用本地插件路径安装。
