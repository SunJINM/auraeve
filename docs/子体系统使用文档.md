# 子体系统使用文档

## 概述

AuraEve 子体系统实现母子监督自治架构，支持本地和远程子体的统一编排。母体（AuraEve）通过 `TaskOrchestrator` 分配任务，子体自主执行并上报结果。

## 架构分层

```
┌─────────────────────────────────────────┐
│              Agent 工具层                │  subagent_task.py
├─────────────────────────────────────────┤
│              控制面                      │  orchestrator / scheduler / approval
├──────────┬──────────┬───────────────────┤
│  本地执行 │  远程传输  │   记忆合并         │
│  executor │ transport │  merge_service    │
├──────────┴──────────┴───────────────────┤
│              数据层                      │  models / repositories / graph_store
└─────────────────────────────────────────┘
```

## 快速开始

### 1. 通过主体工具调用子体

主体可通过 `subagent` 工具管理子体任务：

```
# 派生单个任务
subagent(action="spawn", goal="搜索最新的 AI 论文并总结")

# 查询任务列表
subagent(action="list")

# 查看任务详情
subagent(action="status", task_id="task-xxxx")

# 引导运行中的任务
subagent(action="steer", task_id="task-xxxx", message="重点关注 RAG 方向")

# 暂停/恢复/取消
subagent(action="pause", task_id="task-xxxx")
subagent(action="resume", task_id="task-xxxx")
subagent(action="cancel", task_id="task-xxxx")
```

### 2. 提交 DAG 任务组

```
subagent(action="dag", tasks=[
    {"id": "A", "goal": "收集数据"},
    {"id": "B", "goal": "清洗数据", "depends_on": ["A"]},
    {"id": "C", "goal": "训练模型", "depends_on": ["B"]},
    {"id": "D", "goal": "生成报告", "depends_on": ["C"]},
])
```

任务按依赖拓扑自动调度，A 完成后 B 才开始执行。

### 3. 审批高风险操作

子体执行高风险操作（如 shell 命令、文件写入）时会请求审批：

```
subagent(action="approve", approval_id="apv-xxxx", decision="approve")
subagent(action="approve", approval_id="apv-xxxx", decision="reject")
```

## 本地子体

本地子体运行在母体进程内，作为 `asyncio.Task` 执行，零网络开销。

- 自动注册：`RuntimeKernel` 启动时自动注册 `node_id="local"` 节点
- 通过 `InProcessReporter` 直接调用 `TaskOrchestrator` 方法上报
- 支持暂停/恢复/取消/引导

## 远程子体

### 配置母体

在配置文件中启用远程子体支持：

```python
NODE_ENABLED = True
NODE_HOST = "0.0.0.0"
SUBAGENT_WS_PORT = 9800

NODE_TOKENS = {
    "node-office": {
        "token": "your-secret-token",
        "display_name": "办公室节点",
        "platform": "linux",
    },
}
```

### 启动远程子体

```bash
python -m node_subagent.main \
    --node-id node-office \
    --token your-secret-token \
    --mother-url ws://mother-host:9800 \
    --workspace /path/to/workspace
```

### 连接流程

1. 远程子体发送 `node.connect` 握手消息（含 node_id + token）
2. 母体验证 token，返回 `node.connect.ok` + 待恢复任务列表
3. 进入消息循环，母体下发任务，子体上报进度/结果

### 自动重连

远程子体断连后自动指数退避重连（5s → 10s → 20s → ... → 60s 上限）。

## 任务状态机

```
queued → dispatched → running → completed
                        ↓         ↓
                      paused    failed → compensating → compensated
                        ↓         ↓
                      running   canceled
                        ↓
                    input_required → running
                                    ↓
                                  timed_out
```

终态：`completed` / `failed` / `canceled` / `timed_out` / `compensated`

## 四维风险策略

| 维度 | 说明 |
|------|------|
| L1 工具级 | 每个工具有默认风险等级（shell=HIGH, read_file=LOW） |
| L2 参数级 | 检测危险参数模式（rm -rf, format 等）→ 升级为 CRITICAL |
| L3 节点级 | 生产节点上 MEDIUM 自动升级为 HIGH |
| L4 任务级 | 高优先级任务（≥8）MEDIUM 升级为 HIGH |

风险决策：
- `LOW/MEDIUM`：自动放行
- `HIGH`：需要人工审批
- `CRITICAL`：默认拒绝

## Saga 补偿

任务可携带 `compensate_action`，失败时自动触发补偿：

```
failed → compensating → compensated
```

补偿任务作为独立子体任务执行。

## 知识图谱

子体产生的知识通过 `MemoryDelta` 增量上报，由 `MemoryMergeService` 合并到母体知识图谱：

| Delta 类型 | 合并策略 |
|-----------|---------|
| fact | 冲突检测，高置信度覆盖低置信度 |
| experience | 直接合并为 (task, learned, content) 三元组 |
| observation | 直接合并为 (entity, observed, detail) 三元组 |

## 能力自适应调度

`Scheduler` 根据综合评分选择最优子体：

```
score = 0.4 × capability + 0.35 × (1 - load_ratio) + 0.25 × priority_match
```

- `capability`：历史成功率（由 `CapabilityTracker` 维护）
- `load_ratio`：当前负载 / 最大并发
- `priority_match`：任务优先级匹配

## 分布式追踪

所有操作携带 `trace_id / span_id / parent_span_id`，记录到 `task_events` 表：

```python
# 查看任务追踪链
telemetry_hub.get_trace(trace_id)
```

├── protocol/

├── runtime/
│   ├── react_loop.py      # ReAct 执行循环
│   ├── reporter.py        # 上报接口（InProcessReporter）
│   ├── capabilities.py    # 能力注册表
│   ├── local_memory.py    # 本地记忆存储
│   └── sandbox.py         # 执行沙箱
├── control_plane/
│   ├── orchestrator.py    # 核心编排器
│   ├── scheduler.py       # 自适应调度器
│   ├── approval_center.py # 审批中心
│   ├── policy_v2.py       # 四维风险策略
│   ├── local_executor.py  # 本地子体执行器
│   ├── capability_tracker.py # 能力追踪器
│   └── telemetry_hub.py   # 遥测中心
├── memory/
│   └── merge_service.py   # 记忆合并服务

    ├── auth.py            # Token 认证
    ├── ws_server.py       # WebSocket 服务端（母体侧）

├── main.py                # 远程子体启动入口

└── reporter.py            # WebSocket 上报器

auraeve/agent/tools/
└── subagent_task.py       # 主体工具接口
