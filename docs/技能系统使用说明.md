# AuraEve 技能系统使用说明

## 1. 系统定位

技能（Skill）与插件（Plugin）是两套独立系统：

1. 插件：Python 可执行扩展（工具/Hook/渠道能力）。
2. 技能：`SKILL.md` + metadata，负责能力说明、依赖检查与安装器执行。

## 2. 技能来源与优先级

按名称冲突覆盖顺序：

1. 内置：`auraeve/skills`
2. 插件声明的技能目录（仅启用插件）
3. 托管：`~/.auraeve/skills`
4. 个人 agents：`~/.agents/skills`
5. 项目 agents：`<workspace>/.agents/skills`
6. 工作区：`<workspace>/skills`
7. 额外目录：`SKILLS_LOAD_EXTRA_DIRS`

## 3. 状态与产物目录

1. 技能状态：`~/.auraeve/skills-state.json`
2. 托管技能目录：`~/.auraeve/skills`
3. 下载型安装器产物：`~/.auraeve/tools/<skill-key-hash>`

## 4. CLI 命令

```bash
python main.py skills list --json
python main.py skills info <id> --json
python main.py skills status --json
python main.py skills install <id> [--install-id <installer-id>] --json
python main.py skills enable <skill-key> --json
python main.py skills disable <skill-key> --json
python main.py skills doctor --json
python main.py skills sync [--all] [--dry-run] --json
```

## 5. WebUI API

1. `GET /api/webui/skills/list`
2. `GET /api/webui/skills/info?id=...`
3. `GET /api/webui/skills/status`
4. `POST /api/webui/skills/install` `{ id, installId }`
5. `POST /api/webui/skills/enable` `{ id }`
6. `POST /api/webui/skills/disable` `{ id }`
7. `GET /api/webui/skills/doctor`
8. `POST /api/webui/skills/sync` `{ all, dryRun }`

## 6. 支持的安装器

metadata.install 支持：

1. `brew`
2. `apt`
3. `node`
4. `go`
5. `uv`
6. `download`

安装器选择规则：

1. 指定 `--install-id` 时精确命中。
2. 未指定时按偏好选择（brew 优先可配置），否则回退到可用安装器。

## 7. 安全策略

1. download 仅允许 `http/https`。
2. 可通过 `SKILLS_SECURITY_ALLOWED_DOWNLOAD_DOMAINS` 限制下载域名。
3. 归档解压启用路径越界检查，拒绝链接型条目。
4. `targetDir` 被限制在技能 tools 根目录内。
