# 节点系统使用文档

## 概述

节点系统允许在服务器运行的 auraeve（Eve）控制本地电脑。本地电脑作为"节点"连接服务器，Eve 可以通过工具在本地执行 shell 命令、读写文件、截图等操作，所有记忆和智能体逻辑仍在服务器端。

---

## 快速上手

### 第一步：生成令牌

在服务器上运行：

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

记录输出的令牌（例如 `a1b2c3d4...`）。

---

### 第二步：配置服务器

编辑服务器的 `auraeve/config.py`：

```python
# 启用节点系统
NODE_ENABLED = True
NODE_HOST = "0.0.0.0"   # 监听所有网卡
NODE_PORT = 8765         # 监听端口

# 配置已配对节点
NODE_TOKENS = {
    "home-pc": {
        "token": "a1b2c3d4...",    # 第一步生成的令牌
        "display_name": "家里电脑",
        "platform": "windows",
    },
}
```

重启服务器：

```bash
python main.py
```

日志中出现 `节点服务器：ws://0.0.0.0:8765` 表示启动成功。

---

### 第三步：配置本地节点

在本地电脑安装依赖：

```bash
pip install websockets loguru
# 可选：截图功能
pip install mss Pillow
# 可选：系统信息
pip install psutil
```

从项目根目录复制配置文件：

```bash
cp node_client/config.example.py node_client/config.py
```

编辑 `node_client/config.py`：

```python
SERVER_URL = "ws://your-server-ip:8765"   # 服务器公网 IP 或域名
NODE_ID = "home-pc"                         # 与服务器 NODE_TOKENS 的键一致
TOKEN = "a1b2c3d4..."                       # 与服务器 NODE_TOKENS 中的 token 一致
DISPLAY_NAME = "家里电脑"
```

---

### 第四步：启动本地节点

```bash
# 方式一：使用配置文件（推荐）
python -m node_client

# 方式二：命令行参数
python -m node_client --server ws://your-server:8765 --node-id home-pc --token a1b2c3d4...
```

本地终端显示：
```
09:00:00 | INFO     | 正在连接：ws://your-server:8765
09:00:01 | INFO     | 已连接服务器，节点 ID：home-pc
```

---

### 第五步：通过 Eve 控制本地电脑

现在可以对 Eve 说：

> "帮我看看家里电脑的磁盘使用情况"
> "在家里电脑上执行 `ipconfig` 命令"
> "列出家里电脑桌面的文件"
> "帮我截一张家里电脑的屏幕"

## 断线重连

本地节点内置自动重连机制：
- 初始等待 3 秒重连
- 每次失败后等待时间乘以 1.5（最长 60 秒）
- 服务器端将离线期间的调用入待机队列，节点重连后自动执行

---

## 防火墙配置

确保服务器的 `NODE_PORT`（默认 8765）对本地电脑开放：

```bash
# Ubuntu/Debian
sudo ufw allow 8765/tcp

# CentOS/RHEL
sudo firewall-cmd --permanent --add-port=8765/tcp
sudo firewall-cmd --reload
```

## 添加更多节点

在 `auraeve/config.py` 的 `NODE_TOKENS` 中添加更多条目，每个节点使用独立的令牌：

```python
NODE_TOKENS = {
    "home-pc": {
        "token": "token-for-home-pc",
        "display_name": "家里电脑",
        "platform": "windows",
    },
    "office-laptop": {
        "token": "token-for-office",
        "display_name": "公司笔记本",
        "platform": "macos",
    },
}
```

## 常见问题

**Q：节点连接后立即断开？**
检查 token 是否与服务器配置完全一致，注意大小写和空格。

**Q：命令执行超时？**
默认超时 60 秒，长时间任务可通过 `timeout_ms` 参数调整（最大由服务器端决定）。

**Q：节点离线时 Eve 会怎样？**
Eve 会收到"节点离线，调用已入队"的提示，并将操作加入待机队列。节点上线后自动执行。

**Q：如何安全地暴露端口？**
建议使用 nginx 反向代理并配置 SSL（`wss://`），或通过 VPN 限制访问来源。
