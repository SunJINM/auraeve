# AuraEve CLI 命令说明（按模块）

本文档基于当前实现文件 `auraeve/cli/app.py`，描述 `auraeve` 命令的模块划分、用途、参数与示例。

## 1. 总览

主入口：

- `auraeve ...`
- 或 `python -m auraeve ...`（无需安装脚本时推荐）

行为说明：

- 当不带子命令直接执行 `auraeve` 时，等价于执行 `auraeve run`
- 顶层可用 `--terminal`（仅在无子命令场景生效）

常见退出码：

- `0`：成功
- `1`：业务失败/校验失败
- `2`：参数值格式错误（例如 `config set --strict-json` 解析失败）

## 3. 运行模块

### 3.1 `auraeve run`

作用：

- 启动 AuraEve 主服务运行时（Agent + 渠道 + WebUI + 插件/技能 + 定时任务等）

参数：

- `--terminal`：终端模式运行（本地调试）

示例：

- `auraeve run`
- `auraeve run --terminal`

前置校验：

- 会检查配置文件是否存在且合法
- 必填项（当前是 `LLM_API_KEY`）必须非空

---

## 4. 健康与诊断模块

### 4.1 `auraeve health`

作用：

- 检查 Executor 健康状态（`/api/executor/health`）

参数：

- `--url`：Executor 地址（默认从 `EXECUTOR_URL` 或配置读取）
- `--token`：Executor Token（默认从 `EXECUTOR_TOKEN` 或配置读取）
- `--json`：JSON 输出

示例：

- `auraeve health`
- `auraeve health --url http://127.0.0.1:18791 --token xxx --json`

### 4.2 `auraeve status`

作用：

- 汇总基础状态：配置、默认 workspace、executor、插件数量、技能数量

参数：

- `--json`：JSON 输出

示例：

- `auraeve status`
- `auraeve status --json`

### 4.3 `auraeve doctor`

作用：

- 聚合诊断（配置 + 插件 + 技能）

参数：

- `--fix`：仅对配置诊断尝试自动修复
- `--json`：JSON 输出

示例：

- `auraeve doctor`
- `auraeve doctor --fix --json`

说明：

- `--fix` 只传递给配置医生（`config doctor`）
- 插件和技能诊断为只读检查

---

## 5. Executor 模块

子命令：`auraeve executor <subcommand>`

### 5.1 `auraeve executor start`

作用：

- 启动 Host Executor 服务

参数：

- `--host`：绑定地址（默认 `EXECUTOR_HOST` 或配置，最终默认 `0.0.0.0`）
- `--port`：绑定端口（默认 `EXECUTOR_PORT` 或配置，最终默认 `18791`）
- `--token`：覆盖 Token（默认 `EXECUTOR_TOKEN` 或配置）

示例：

- `auraeve executor start`
- `auraeve executor start --host 0.0.0.0 --port 18791 --token xxx`

### 5.2 `auraeve executor health`

作用：

- 与顶层 `health` 一致，检查 Executor 健康

参数：

- `--url`
- `--token`
- `--json`

示例：

- `auraeve executor health --json`

### 5.3 `auraeve executor token`

作用：

- 查看或轮换 Executor Token

参数：

- `--rotate`：生成新 token 并写入配置
- `--json`：JSON 输出

示例：

- `auraeve executor token`
- `auraeve executor token --rotate --json`

## 6. 配置模块（config）

子命令：`auraeve config <subcommand>`

### 6.1 `file`

作用：

- 输出当前配置文件路径

参数：

- `--json`

示例：

- `auraeve config file`
- `auraeve config file --json`

### 6.2 `path`

作用：

- 输出路径解析结果（state/config/agents/workspace/vectordb 等）

参数：

- `--agent`：按指定 agent 输出 workspace 解析（默认 `default`）
- `--explain`：附加 workspace 决策链
- `--json`

示例：

- `auraeve config path`
- `auraeve config path --agent dev --explain --json`

### 6.3 `validate`

作用：

- 校验配置文件合法性

参数：

- `--json`

示例：

- `auraeve config validate`
- `auraeve config validate --json`

### 6.4 `doctor`

作用：

- 配置诊断，可选自动修复

参数：

- `--fix`：自动修复可修复项
- `--json`

示例：

- `auraeve config doctor`
- `auraeve config doctor --fix --json`

### 6.5 `get`

作用：

- 读取指定配置项值

参数：

- `key`：配置键（位置参数）
- `--json`

示例：

- `auraeve config get LLM_MODEL`
- `auraeve config get WEBUI_ENABLED --json`

### 6.6 `set`

作用：

- 写入配置项

参数：

- `key`：配置键（位置参数）
- `value`：配置值（位置参数）
- `--strict-json`：要求 `value` 必须是合法 JSON

参数细节：

- 不加 `--strict-json` 时会尝试 JSON 解析，解析失败则按原始字符串写入
- 加 `--strict-json` 时解析失败直接返回错误（退出码 2）

示例：

- `auraeve config set LLM_MODEL "\"gpt-4o-mini\"" --strict-json`
- `auraeve config set EXECUTOR_URL http://127.0.0.1:18791`

### 6.7 `unset`

作用：

- 删除指定配置键（回退默认）

参数：

- `key`：配置键（位置参数）

示例：

- `auraeve config unset LLM_MODEL`

## 7. 工作区模块（workspace）

子命令：`auraeve workspace <subcommand>`

### 7.1 `resolve`

作用：

- 解析指定 agent 的 workspace 路径与决策来源

参数：

- `--agent`：agent ID（默认 `default`）
- `--json`

示例：

- `auraeve workspace resolve`
- `auraeve workspace resolve --agent dev --json`

---

## 8. 插件模块（plugins）

子命令：`auraeve plugins <subcommand>`

统一说明：

- 此模块会先做配置有效性检查（配置不存在/非法会先报错）
- workspace 使用 `default` agent 对应目录

### 8.1 `list`

作用：

- 列出插件

参数：

- `--json`

示例：

- `auraeve plugins list`
- `auraeve plugins list --json`

### 8.2 `info`

作用：

- 查看单个插件详情

参数：

- `plugin_id`：插件 ID（位置参数）
- `--json`

示例：

- `auraeve plugins info demo.local`

### 8.3 `install`

作用：

- 安装插件（目录或包）

参数：

- `path_input`：插件路径（位置参数）
- `--link`：软链接安装模式

示例：

- `auraeve plugins install ./tmp_plugin_demo`
- `auraeve plugins install ./tmp_plugin_demo --link`

### 8.4 `uninstall`

作用：

- 卸载插件

参数：

- `plugin_id`：插件 ID（位置参数）
- `--keep-files`：卸载后保留文件

示例：

- `auraeve plugins uninstall demo.local`
- `auraeve plugins uninstall demo.local --keep-files`

### 8.5 `enable` / `disable`

作用：

- 启用/禁用插件

参数：

- `plugin_id`：插件 ID（位置参数）

示例：

- `auraeve plugins enable demo.local`
- `auraeve plugins disable demo.local`

### 8.6 `doctor`

作用：

- 插件健康检查

参数：

- `--json`

示例：

- `auraeve plugins doctor --json`

---

## 9. 技能模块（skills）

子命令：`auraeve skills <subcommand>`

统一说明：

- 此模块会先做配置有效性检查
- workspace 使用 `default` agent 对应目录

### 9.1 `list`

作用：

- 列出技能

参数：

- `--json`

### 9.2 `info`

作用：

- 查看技能详情

参数：

- `skill`：技能 key 或名称（位置参数）
- `--json`

### 9.3 `status`

作用：

- 输出技能状态报告

参数：

- `--json`

### 9.4 `install`

作用：

- 执行技能依赖安装流程

参数：

- `skill`：技能 key 或名称（位置参数）
- `--install-id`：指定安装器 ID（例如 brew/apt 等）
- `--json`

### 9.5 `enable` / `disable`

作用：

- 启用/禁用技能

参数：

- `skill_key`：技能 key（位置参数）
- `--json`

### 9.6 `doctor`

作用：

- 技能诊断（依赖、可执行环境等）

参数：

- `--json`

### 9.7 `sync`

作用：

- 同步技能状态/索引

参数：

- `--all`：同步全部技能
- `--dry-run`：只预览，不执行
- `--json`

示例：

- `auraeve skills sync`
- `auraeve skills sync --all --dry-run --json`

---

## 10. 部署模块（deploy）

子命令：`auraeve deploy <subcommand>`

说明：

- 该模块本质是转调 `deploy/` 目录下脚本
- Windows 调用 PowerShell 脚本，Linux/macOS 调用 bash 脚本

### 10.1 `one-click`

作用：

- 执行一键部署脚本

参数：

- `mode`：`auto|docker|local`（默认 `auto`）

示例：

- `auraeve deploy one-click`
- `auraeve deploy one-click docker`

### 10.2 `update`

作用：

- 执行更新脚本（拉代码并重启）

参数：

- `mode`：`auto|docker|local`（默认 `auto`）

示例：

- `auraeve deploy update`
- `auraeve deploy update local`

---

## 11. 常见组合示例

- 首次检查配置：`auraeve config file && auraeve config validate`
- 本地启动主服务：`auraeve run --terminal`
- 启动执行器并检查：`auraeve executor start --host 0.0.0.0 --port 18791` + `auraeve executor health --json`
- 总体巡检：`auraeve doctor --json`
- 插件巡检：`auraeve plugins list --json` + `auraeve plugins doctor --json`
- 技能巡检：`auraeve skills status --json` + `auraeve skills doctor --json`
